模块化明细功能方案

高度模块化设计，支持自定义显示和多维度明细查询

一、方案概述

本方案将明细功能拆分为三个核心模块：

ConfigModule

配置管理模块，负责读取和提供配置数据

DetailModule

明细逻辑模块，负责执行明细查询

UIModule

UI渲染模块，负责渲染可点击单元格和弹窗

{
    "id": "rad_workload",
    "name": "影像中心工作量",
    "procName": "usp_TJFX_RadiologyWorkload",
    "templateName": "rad_workload.html",
    
    "detail": {
        "enabled": true,
        "triggerMode": "multi",
        "triggerField": "医生姓名",
        "excludeFields": ["合计"],
        "customFields": [],
        "inheritParams": ["@StartDate", "@EndDate", "@StatisticsType"],
        "customRenderer": null
    }
}
配置项说明
配置项 说明 示例
enabled 是否启用明细功能 true / false
triggerMode 触发模式 single / multi / custom
triggerField 单字段模式的触发字段 医生姓名
excludeFields 多字段模式的排除字段 ["合计", "序号"]
inheritParams 继承的参数列表 ["@StartDate", "@EndDate"]
customRenderer 自定义渲染函数名 myCustomRenderer
四、触发模式说明

单字段触发

只有指定的一个字段可点击，适用于只需医生明细的场景

多字段触发

所有数值列都可点击（排除指定字段），适用于全方位明细查询

自定义触发

只有指定的多个字段可点击，提供精细控制

五、自定义显示机制

通过注册全局渲染函数实现自定义显示：

window.customDetailRenderer = function(data) {
    return `
        

            
检查明细

            

                总记录数: ${data.length}
            

            

                ${data.map(item => `
                    
${item.检查日期} - ${item.患者姓名}

                `).join('')}
            

        

    `;
};
